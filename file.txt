1. Початок роботи
git init - Ініціалізація Git-репозиторію (локально)
Створює приховану папку .git, яка містить всю інформацію про репозиторій.

2. Клонування існуючого репозиторію
git clone <URL> - Завантажує репозиторій з GitHub, GitLab або іншого сервера.

3. Статус папки
git status - Перевірка статусу файлів: Показує зміни у файлах (нові, змінені, видалені) та їх стан у Git.

4. Додавання файлів у індекс (стейджинг) - Готує файли до коміту.
git add file.txt  # Додати конкретний файл
git add .         # Додати всі зміни в поточній папці

5. Коміт (збереження змін у Git)
git commit -m "Опис змін" - Фіксує зміни локально.
